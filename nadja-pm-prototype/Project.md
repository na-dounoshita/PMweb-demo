# PROJECT.md

## 概要

NADJA Process Intelligence Platform の MVPプロトタイプ。ProcessMiningTest（C# WinFormsアプリ）が出力したCSVをアップロードし、プロセスマップ（DFG）と基本KPIをブラウザで表示するWebアプリ。`docker compose up` で3コンテナ（PostgreSQL / FastAPI / Streamlit）が起動する。

## 技術スタック

- **言語**: Python 3.11
- **DB**: PostgreSQL 16（JSONB対応）
- **API**: FastAPI + Uvicorn
- **分析エンジン**: pm4py 2.7
- **フロントエンド**: Streamlit + Plotly + vis.js (CDN直接読み込み)
- **コンテナ**: Docker Compose（3サービス）

## 起動・停止

**前提条件:** Docker Desktop が起動していること

```bash
cd nadja-pm-prototype

# 起動（初回はビルドに数分かかる）
docker compose up --build

# 停止
docker compose down

# データも含めて完全リセット
docker compose down -v
```

| サービス | URL | 用途 |
|---------|-----|------|
| Streamlit | http://localhost:8501 | メインUI |
| FastAPI | http://localhost:8000/docs | API確認（Swagger） |
| PostgreSQL | localhost:5432 | DB（直接アクセスは不要） |

**使い方:**

1. http://localhost:8501 を開く
2. 「CSVアップロード」ページでプロセス名を入力し、CSVファイルをアップロード
3. 「プロセスマップ」ページでDFGを確認（ウィンドウレベル/タスクレベル切替可能）
4. 「タスクマイニング」ページでイベントに業務タスク名をタグ付け
5. 「KPIダッシュボード」ページで指標・グラフを確認

## 画面操作ガイド

アプリは **5ページ構成**。左サイドバーのラジオボタンでページを切り替える。

### 全体の操作フロー

```
① CSVアップロード → ② プロセスマップでDFG確認 → ③ タスクマイニングで業務タスクを定義・タグ付け → ④ プロセスマップで「タスクレベル」DFG確認 → ⑤ KPIダッシュボードで数値確認
```

### サイドバー（共通）

| 要素 | 動作 |
|------|------|
| ラジオボタン「CSVアップロード」 | アップロード画面に切り替え |
| ラジオボタン「プロセスマップ」 | DFG可視化画面に切り替え |
| ラジオボタン「タスクマイニング」 | タスク定義・イベントタグ付け画面に切り替え |
| ラジオボタン「プロセスマップα」 | 手動編集可能なマップ画面に切り替え |
| ラジオボタン「KPIダッシュボード」 | KPI画面に切り替え |

### ページ1: CSVアップロード

| 操作 | 何が起きるか |
|------|-------------|
| 「CSVファイルを選択」にファイルをドラッグ or クリック | CSVファイルが読み込まれる（`.csv` のみ受付）。ヘッダーを自動解析し、CaseIDカラムの有無を検出 |
| CaseIDカラムが無い場合 → 「タイムギャップ閾値（分）」入力 | 1〜1440分の範囲で設定（デフォルト30分、ステップ5分）。この時間以上の間隔があると新しいケースとして分割 |
| CaseIDカラムがある場合 → info表示 | 既存のケースIDを使用する旨を表示 |
| 「プロセス名」テキスト欄に入力 | 分析対象のプロセス名を設定（例: `営業事務`） |
| **「アップロード」ボタン** | ファイルとプロセス名の**両方が揃うまでグレーアウト**。押すとCSVをAPIに送信→PostgreSQLに格納。成功時は「インポートイベント数」と「ケース数」を表示。失敗時はエラーメッセージを表示 |

### ページ2: プロセスマップ（DFG）

| 操作 | 何が起きるか |
|------|-------------|
| ページを開く | 登録済みプロセスを自動取得。未登録なら「先にCSVをアップロードしてください」と案内 |
| **「プロセス選択」ドロップダウン** | 選択するとDFG（Directly-Follows Graph）を**自動描画**。ボタン操作は不要 |
| マウスホイール | ズームイン / ズームアウト |
| 背景ドラッグ | パン（表示領域の移動） |
| ノードドラッグ | ノードの配置を手動調整 |
| ノード/エッジにホバー | 詳細ツールチップを表示 |

**サイドバー フィルタ設定:**

| 要素 | 動作 |
|------|------|
| 「最小頻度フィルタ」スライダー | 指定値未満の頻度のエッジを非表示。主要フローだけを見たい場合に使う |
| 「所要時間を表示」チェック | エッジラベルに平均所要時間を表示/非表示 |
| 「クリティカルパス強調」チェック | 最高頻度の経路をハイライトし、他のエッジを薄く表示 |
| 「表示レベル」ラジオ | 「ウィンドウレベル」= 従来のDFG、「タスクレベル」= タスクマイニングでタグ付けした業務タスク単位のDFG |

**DFGの見方:**

| 要素 | 意味 |
|------|------|
| 四角（ノード） | アクティビティ名と実行回数。大きいほど実行回数が多い |
| 矢印（エッジ） | アクティビティ間の遷移。太いほど頻度が高い |
| 矢印のラベル | 遷移回数 + 平均所要時間（○分○秒形式） |
| エッジの色 | 青=速い → 黄=中間 → 赤=遅い（ボトルネック箇所が赤で一目瞭然） |
| 緑ノード | 開始アクティビティ |
| 赤ノード | 終了アクティビティ |
| 橙ノード | 開始かつ終了アクティビティ |
| 青灰ノード | 中間アクティビティ |

### ページ3: タスクマイニング

2タブ構成でタスク定義とイベントタグ付けを行う。

**タブ1: タスク定義**

| 操作 | 何が起きるか |
|------|-------------|
| タスク名・説明・色を入力して「作成」 | 業務タスク定義をDBに登録（例: 受注処理、見積作成） |
| 「削除」ボタン | タスク定義を削除（紐づくタグも連動削除） |

**タブ2: イベントタグ付け**

| 操作 | 何が起きるか |
|------|-------------|
| ケース選択 | そのケースのイベント一覧を色付きテーブルで表示。タグ済みイベントは色が付く |
| 開始行・終了行を指定してタスク選択 → 「タグ付け」 | 指定範囲のイベントに業務タスク名を付与。重複チェックあり |
| タグ済み一覧の「解除」ボタン | タグを外す |
| タグ率メトリクス | ケース内のタグ付け進捗を表示 |

タグ付け後、「プロセスマップ」ページで「タスクレベル」に切り替えると業務レベルDFGが表示される。

### ページ4: プロセスマップα

| 操作 | 何が起きるか |
|------|-------------|
| ページを開く | 登録済みプロセスを自動取得。サイドバーでプロセスを選択 |
| **「DFGをマップとして保存」ボタン** | 自動生成DFGを編集可能なマップとしてDBに保存 |
| **マップ選択ドロップダウン** | 保存済みマップを選択して編集キャンバスに表示 |
| vis.jsツールバー「ノード追加」 | キャンバスをクリックしてノード名を入力し配置 |
| vis.jsツールバー「エッジ追加」 | ノード間をドラッグして遷移を作成 |
| ノードをダブルクリック | ノード名を変更 |
| 選択してDeleteキー or ツールバー「削除」 | ノード/エッジを削除 |
| **「💾 保存」ボタン（キャンバス内）** | 編集内容をFastAPI経由でDBに保存 |
| **「JSONエクスポート」/「CSVエクスポート」** | マップをファイルとしてダウンロード |
| **「JSONインポート」/「CSVインポート」** | ファイルからマップを読み込み。CSV形式は `From,To,Label` 列 |

### ページ5: KPIダッシュボード

| 操作 | 何が起きるか |
|------|-------------|
| ページを開く | 登録済みプロセスを自動取得。未登録なら「先にCSVをアップロードしてください」と案内 |
| **「プロセス選択」ドロップダウン** | 選択すると以下4セクションを**自動表示**。ボタン操作は不要 |

**表示される4セクション:**

| セクション | 内容 |
|-----------|------|
| メトリクスカード（上部） | 「ケース数」「平均所要時間（○分○秒）」「バリアント数」の3指標 |
| 棒グラフ（中部） | アクティビティ別の実行回数（横軸=アクティビティ名、縦軸=回数） |
| 円グラフ（中下部） | アクティビティ別の合計時間割合（時間データがある場合のみ表示） |
| バリアント一覧（下部） | 業務パターン一覧テーブル（パターン名・件数・割合%・平均所要時間） |

## メモ

- CSVにCaseIDカラムが無い場合、タイムギャップ閾値（分単位、UI上で選択可能）でケースIDを自動生成。閾値を超える間隔があると新しいケースとして分割
- CSVの固定3カラム（CaseID, Activity, Timestamp）のうち、CaseIDは任意。Activity, Timestamp以外は `event_attrs JSONB` に格納。カラム追加時にスキーマ変更不要
- 同一プロセスへの再アップロードは既存データを削除して上書き（プロトタイプ仕様）
- Streamlitコンテナにpm4pyは不要。APIが返すJSONからvis.js (CDN) でインタラクティブグラフを描画
- PostgreSQLの `healthcheck` + `condition: service_healthy` でAPI起動順序を保証
